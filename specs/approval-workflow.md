## Application context

The Approval Portal is a document management application where users submit
documents for review. It supports two personas:

- **Submitter (Alex Chen)** — can submit documents and track their status
- **Approver (Morgan Lee)** — can review, approve, or reject pending documents

The app persists state to `localStorage`. Each browser session is pre-authenticated
via the session storage state files in `.auth/`.

### 2. Submit a new document

**Persona:** Alex Chen (submitter)
**Seed:** `tests/seed.spec.ts`

#### Steps
1. Click the **+ Submit document** button
2. Observe the submit modal opens
3. Fill in "Q1 Compliance Report" in the title field
4. Select "Legal" from the category dropdown
5. Fill in a description: "Annual compliance summary for Q1."
6. Select "High" priority
7. Click **Submit for approval**

#### Expected results
- Modal closes
- A success toast notification appears
- The new document "Q1 Compliance Report" appears at the top of the document list
- The document shows status "pending"
- The Pending stats counter increments by 1

#### Edge cases

- **Empty title**: Click submit without a title — browser validation prevents submission, title field receives focus
- **No category selected**: Form requires a category — submission blocked until selected
- **Cancel**: Clicking Cancel or the backdrop closes the modal without adding a document
