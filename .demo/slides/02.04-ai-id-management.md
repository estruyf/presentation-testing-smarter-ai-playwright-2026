# Create a contract between your app and your tests

---

# Your first line of defence

CSS classes change. A `data-testid` has **one job** — connecting your app to your test suite.

But only if you name them well:

| ❌ Avoid | ✅ Prefer |
|---|---|
| `data-testid="btn1"` | `data-testid="submit-document-button"` |
| `data-testid="input2"` | `data-testid="document-title-input"` |
| `class="badge"` | `data-testid="approval-status-badge"` |

**Be descriptive. Include context. Make the intent obvious** — to humans reading the test *and* to AI generating or healing it.

> Good test IDs are the cheapest investment you can make in test stability. This is a great task for AI to help you.

# Let AI add your test IDs

> *"Analyse the website in `./apps/approval-portal` and check which components are important for end-to-end testing. Add `data-testid` attributes to those components."*

- Adds descriptive `data-testid` attributes following your naming conventions

# One step further: a shared constants file

> *"Add all the retrieved and created test IDs in a TypeScript constant file under `apps/approval-portal/src/constants/testIds.ts`. That way, it's easier to reuse in E2E tests later."*

```typescript
// apps/approval-portal/src/constants/testIds.ts
export const TestIds = {
  searchInput: "search-input",
  docTitle: "doc-title",
  docCategory: "doc-category",
  btnSubmitForm: "btn-submit-form",
  documentRow: "document-row",
  btnReview: "btn-review",
  btnApprove: "btn-approve",
  btnReject: "btn-reject",
  reviewComment: "review-comment",
  approvalStatusBadge: "doc-status",
};
```

# Using the constants in your tests

The constants file bridges your app and your test suite.

```typescript
import { TestIds } from "../constants/testIds";

// In your Playwright tests
await page.getByTestId(TestIds.docTitle).fill("Q1 Compliance Report");
await page.getByTestId(TestIds.docCategory).selectOption("legal");
await page.getByTestId(TestIds.btnSubmitForm).click();
```

**The workflow:**

1. 🤖 AI scans the app and adds `data-testid` attributes
2. 🤖 AI generates a shared constants file
3. ✅ Tests reference constants — not hardcoded strings
4. 🔒 Refactors stay safe — change the ID in one place
